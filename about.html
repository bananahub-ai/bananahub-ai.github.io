About BananaHub

A template hub for people first, with stable machine access when needed.

BananaHub is the public gallery for installable Nano Banana prompt templates. The homepage is optimized for browsing, while this page holds the background, contribution context, and machine-readable entry points.

Open gallery Template System Template Format

Agents should prefer `llms.txt`, `catalog.json`, and `agent-catalog.md`. Humans should usually start in the gallery.

What BananaHub Is

A lightweight hub for installable prompt modules.

BananaHub is not meant to be one giant prompt dump. Templates remain separate modules that can live in GitHub, appear in the catalog, and be installed only when they are actually useful.

01

People should judge from the output first

The visual gallery exists so users can scan results quickly. Titles, tags, and install commands support that decision, but should not overpower it.

02

Templates stay hot-pluggable

Nano Banana Skill stays lean when prompt modules can be discovered and installed on demand instead of bundled into the base skill up front.

03

Machine-readable files stay first-class

Agents do not need to scrape the website UI. Stable files remain available for structured discovery, ranking, linking, and deterministic installation.

Machine Files

Contribution Loop

BananaHub indexes template repos without becoming the repo itself.

For template authors

Publish self-describing template folders in GitHub, follow the Nano Banana template format, and get indexed through the catalog source. Your templates stay in your repo; BananaHub stays lightweight.

Read the template spec

For advanced users and agents

Use the catalog to detect a likely match, inspect the source template when needed, and install only the exact prompt module that improves the current task.
